# Проверочный лист безопасности бизнес-систем PropDevelopment

## Обоснование выбора разделов

При составлении проверочного листа были выбраны следующие разделы, исходя из проблем и приоритетов компании:

1. **Управление доступом и идентификация** — несколько точек регистрации клиентов без координации, отсутствие единых политик доступа к данным.
2. **Безопасность API и интеграций** — API партнёров (управляющих компаний) используются без единых политик безопасности; партнёры могут получать данные других УК.
3. **Защита персональных данных** — клиенты видят чужие данные в ЛК; риск нарушения 152-ФЗ; биометрические данные (умный домофон).
4. **Управление данными и потоками** — нет системного контроля внутренних потоков данных; проблема блокирует развитие BI/ML/AI.
5. **Аудит и мониторинг** — специалист по ИБ не получает все решения и изменения; нет системного контроля.
6. **Безопасность инфраструктуры** — гибридная инфраструктура (on-premise + облако); Kubernetes-кластеры.
7. **Соответствие законодательству** — неуверенность в соответствии текущих бизнес-процессов требованиям законодательства РФ.

> **Вывод:** Уровень зрелости информационной безопасности компании PropDevelopment критически низкий. Наиболее острые проблемы — отсутствие контроля доступа к данным, незащищённые API партнёров и нарушения при обработке персональных данных. Требуется немедленное проведение комплексного аудита и разработка дорожной карты устранения выявленных несоответствий.
